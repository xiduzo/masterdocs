Components are Figma's reusable building blocks. Define a main component once and use instances everywhere — changes propagate automatically.

## Create and use components

Build your first main component and place instances. Learn overrides, detaching, naming conventions, and organizing components on dedicated pages.

- [Guide to components](https://help.figma.com/hc/en-us/articles/360038662654) — Full reference: main components, instances, overrides, and best practices.
- [Create components to reuse in designs](https://help.figma.com/hc/en-us/articles/360038663154) — Step-by-step creation guide.
- [Name and organize components](https://help.figma.com/hc/en-us/articles/360038663994) — Slash naming for organized libraries.

## Variants

Group related states (primary / secondary / ghost, small / medium / large, default / hover / disabled) into a single component set with variant properties.

- [Create and use variants](https://help.figma.com/hc/en-us/articles/360056440594) — Multi-dimensional variant properties.
- [Create interactive components with variants](https://help.figma.com/hc/en-us/articles/360061175334) — Hover states, toggles, and accordions in prototypes.

## Component properties and slots

Add boolean (show/hide), text, and instance swap properties for lightweight customization without needing full variants. Slots let consumers drop any content inside a component.

<YouTube id="UjOVRUSdO14" />

- [Component properties](https://help.figma.com/hc/en-us/articles/5579474826519) — Boolean, text, and instance swap properties.
- [Use slots to build flexible components](https://help.figma.com/hc/en-us/articles/38231200344599) — Composable component design with slots.
- [Slots vs instance swaps vs variants](https://help.figma.com/hc/en-us/articles/38741465279895) — When to use which approach.

## Overrides and instance management

Customize instances with text, color, visibility, and layout overrides while keeping them linked to the main component. Push overrides back or reset them.

- [Guide to components](https://help.figma.com/hc/en-us/articles/360038662654) — Overrides section covers applying, resetting, and pushing overrides.
