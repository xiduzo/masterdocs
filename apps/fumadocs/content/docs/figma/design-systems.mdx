A design system in Figma is a library file containing your foundations (colors, typography, spacing, icons), reusable components, and documentation. Once published, your team enables the library and uses instances across all their files.

## Define foundations

Establish color palettes, typography scales, spacing tokens, elevation, and icon sets as the base layer of your system.

  - [Lesson 1: Welcome to design systems](https://help.figma.com/hc/en-us/articles/14552802134807) — What is a design system, auditing your product.
  - [Lesson 2: Define your design system](https://help.figma.com/hc/en-us/articles/14552740206743) — Principles, foundations, accessibility, documentation.
- [Habitz design system](https://www.figma.com/community/file/1243224504456122927) — Companion Community file for the course.

## Publish and manage a team library

Publish your file as a team library so others can use it. Review and accept updates, and use branching to test changes before publishing.

- [Publish styles and components](https://help.figma.com/hc/en-us/articles/360025508373) — Publish a file as a team library.
- [Manage library updates](https://help.figma.com/hc/en-us/articles/360039234193) — Review and accept library updates in consumer files.
- [Branching](https://help.figma.com/hc/en-us/articles/360063144053) — Use branches to test library changes before publishing.
